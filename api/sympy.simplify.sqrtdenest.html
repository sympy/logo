def sqrtdenest(expr):

Denests an expression that contains nested square roots. This algorithm is based on <http://www.almaden.ibm.com/cs/people/fagin/symb85.pdf>.

def denester(nested):

Denests a list of expressions that contain nested square roots. This method should not be called directly - use 'denest' instead. This algorithm is based on <http://www.almaden.ibm.com/cs/people/fagin/symb85.pdf>.

It is assumed that all of the elements of 'nested' share the same bottom-level radicand. (This is stated in the paper, on page 177, in the paragraph immediately preceding the algorithm.)

When evaluating all of the arguments in parallel, the bottom-level radicand only needs to be denested once. This means that calling denester with x arguments results in a recursive invocation with x+1 arguments; hence denester has polynomial complexity.

However, if the arguments were evaluated separately, each call would result in two recursive invocations, and the algorithm would have exponential complexity.

This is discussed in the paper in the middle paragraph of page 179.
